# SASRec 团队复现使用指南

按顺序完成以下步骤，即可在本机复现训练与评估流程。

---

## 1. 获取代码

```bash
git clone <你的仓库地址>
cd SASRec
```

本目录已包含 [`sasrec_core/`](sasrec_core/)，notebook 会自动将 `SASREC_DIR` 加入 `sys.path`，无需依赖上级仓库。

---

## 2. 安装依赖

```bash
pip install -r requirements.txt
```

**GPU（推荐）**：先按 [PyTorch 官网](https://pytorch.org/get-started/locally/) 安装与你 CUDA 版本匹配的 `torch`，再安装其余依赖。

验证：

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```

---

## 3. 准备数据

### 方式 A：从维护者已有的 `SASRec_cache` 复制（推荐）

在 `SASRec/` 目录执行：

```bash
python scripts/copy_data_from_cache.py
```

将复制到 `data/`：

- `train.parquet`, `valid.parquet`, `test.parquet`, `item2idx_mapping.parquet`
- `baseline/baseline_sasrec_20260425_130830.json`

### 方式 B：手动复制

从维护者提供的 `SASRec_cache/`（或网盘）将上述 4 个 parquet 拷到 `data/`。

### 校验

```bash
python -c "from pathlib import Path; d=Path('data'); print([p.name for p in d.glob('*.parquet')])"
```

应列出 4 个文件。详见 [data/README.md](data/README.md)。

---

## 4. 全量训练（必做）

1. 启动 Jupyter，工作目录设为 `SASRec/` 或 `SASRec/notebooks/` 均可。
2. 打开 [`notebooks/01_full_train.ipynb`](notebooks/01_full_train.ipynb)。
3. 依次运行所有单元。

**流程概要**：

1. 自动解析 `SASREC_DIR` / `cache_dir`（`data/`）
2. 校验 parquet 是否存在
3. （可选）删除旧 `data/memmap_cache/` 与旧模型
4. `build_memmap_cache` → 生成 `data/memmap_cache/`
5. `SASRecEstimator.fit(...)` 训练
6. `evaluate` 输出 HR@10、NDCG@10
7. 保存 `data/sasrec_full_memmap.pt`

**耗时参考**：GPU 上全量 8 epoch 可能需要数小时；CPU 会自动使用更少 epoch，仅适合冒烟测试。

**与 Baseline 对比**：打开 `data/baseline/baseline_sasrec_20260425_130830.json` 中的 `metrics` 字段。

---

## 5. 网格搜索（可选）

打开 [`notebooks/02_grid_search.ipynb`](notebooks/02_grid_search.ipynb)。

- 默认 `FAST_MODE = True`，评估约 10 万用户，耗时较低。

- 全量复核：将 `FAST_MODE = False`，并缩小 `grid_space` 为 Top 几组参数。

## 8. 常见问题

### `ModuleNotFoundError: No module named 'sasrec_core'`

- 确认 `SASRec/sasrec_core/` 目录存在（9 个源码/文档文件）。
- 先运行 notebook 第一个 code cell，确认 `SASREC_DIR` 指向本目录。
- 若在 monorepo 开发后更新了根目录包，可运行：`python scripts/sync_sasrec_core.py --force`

### `FileNotFoundError: 缺少训练输入文件`

- 执行 `python scripts/copy_data_from_cache.py` 或手动复制 parquet 到 `data/`。

### CUDA OOM（显存不足）

在 `01_full_train.ipynb` 中调小：

- `batch_size`
- `maxlen`
- `hidden_units` / `num_blocks`

或设置 `eval_user_limit=50000` 降低评估开销。

### 指标明显低于 Baseline

- 是否只跑了 1 个 epoch 或用了 CPU 默认少轮次？
- `eval_user_limit` 是否过小导致波动？
- 是否误删/未重建 `memmap_cache`？

### checkpoint 中无 memmap_dir（03 notebook）

- 使用本复现包 `01` 训练保存的 `sasrec_full_memmap.pt`；旧 checkpoint 可能不兼容。
