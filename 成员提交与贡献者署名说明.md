# 成员提交与贡献者署名说明

本文说明：网格搜索跑完后，如何把 `csv` / `jsonl` 提交进仓库，以及如何在 GitHub **Contributors（贡献者）** 页面显示成员姓名。

适用仓库：<https://github.com/kamineayaka/UserBehavior-SASRec>

相关文档：

- 任务与超参：[团队网格搜索任务.md](团队网格搜索任务.md)
- 文件命名：[results/grid_search/README.md](results/grid_search/README.md)

---

## 1. 提交什么

将重命名后的两个文件放入 **`results/grid_search/`**（不要修改 `history/`）：

```
grid_search_fast_{姓名}_{任务ID}_{YYYYMMDD_HHMMSS}.csv
grid_search_fast_{姓名}_{任务ID}_{YYYYMMDD_HHMMSS}.jsonl
```

示例：

```
results/grid_search/grid_search_fast_张三_B1_20260604_153000.csv
results/grid_search/grid_search_fast_张三_B1_20260604_153000.jsonl
```

**不要**提交：`data/*.parquet`、`.pt`、`memmap_cache/` 等大文件（已在 `.gitignore` 中）。

---

## 2. 方式 A：Fork + PR（推荐，贡献者署名最清晰）

### 2.1 成员：Fork 并配置 Git 身份

1. 打开 <https://github.com/kamineayaka/UserBehavior-SASRec>，点击 **Fork**。
2. 在本机配置 Git（**邮箱必须是该成员 GitHub 账号里已验证的邮箱**）：

```bash
git config user.name "张三"
git config user.email "zhangsan@example.com"
```

在 GitHub → **Settings → Emails** 可查看/添加已验证邮箱。

### 2.2 成员：Clone、放文件、推送分支

```bash
git clone https://github.com/你的GitHub用户名/UserBehavior-SASRec.git
cd UserBehavior-SASRec

# 将 csv、jsonl 复制到 results/grid_search/

git checkout -b grid-search-张三-B1

git add results/grid_search/grid_search_fast_张三_B1_*.csv
git add results/grid_search/grid_search_fast_张三_B1_*.jsonl

git commit -m "grid search: B1 by 张三"
git push origin grid-search-张三-B1
```

### 2.3 成员：创建 Pull Request

1. 打开自己 Fork 的仓库页面。
2. 点击 **Compare & pull request**（或 **Pull requests → New pull request**）。
3. 设置：
   - **base repository**：`kamineayaka/UserBehavior-SASRec`，分支 `main`
   - **head repository**：你的 Fork，分支 `grid-search-张三-B1`
4. 标题示例：`grid search: B1 (张三)`
5. 正文建议写明：任务 ID、主要超参、`valid_ndcg10`、GPU/CPU。
6. 点击 **Create pull request**。

### 2.4 维护者：合并 PR（决定 Contributors 是否显示成员）

在 PR 页面点击 **Merge pull request** 时：

| 合并方式 | Contributors 是否显示成员 |
|----------|---------------------------|
| **Create a merge commit**（推荐） | 是 |
| **Rebase and merge** | 是 |
| **Squash and merge** | 通常只显示维护者 |

**请维护者使用「Create a merge commit」**，以保留成员的 commit 作者信息。

合并后，仓库 **Insights → Contributors** 会统计该成员（邮箱需与 GitHub 账号匹配，可能有数小时延迟）。

---

## 3. 方式 B：共享盘 / 群内发给维护者

成员将按命名规范打包的 `csv` + `jsonl` 发给维护者。

维护者放入 `results/grid_search/` 后自行 commit。此时 **默认不会在 Contributors 显示成员**，除非使用下方 **Co-authored-by**。

---

## 4. 维护者代提交时如何署名（Co-authored-by）

若由维护者代为 `git commit`，在 commit message 末尾添加：

```
grid search: B1 by 张三

Co-authored-by: 张三 <zhangsan@example.com>
```

邮箱仍须为成员 GitHub 已验证邮箱。合并到 `main` 后，GitHub 会将该成员记为共同作者。

---

## 5. 方式 C：添加为 Collaborator 后直接 Push

维护者在仓库 **Settings → Collaborators** 邀请成员后，成员可：

```bash
git clone https://github.com/kamineayaka/UserBehavior-SASRec.git
cd UserBehavior-SASRec
git checkout -b grid-search-张三-B1
# add + commit + push
git push origin grid-search-张三-B1
```

再在 GitHub 上向 `main` 开 PR（或由维护者直接合并其分支）。commit 作者为成员本人时，Contributors 会正常统计。

---

## 6. 维护者收到结果后的检查清单

- [ ] 文件在 `results/grid_search/`，未改动 `history/`
- [ ] 命名符合 `grid_search_fast_{姓名}_{任务ID}_{时间}.csv/jsonl`
- [ ] 同时有 **csv** 与 **jsonl**
- [ ] PR 中无 parquet、模型权重等大文件
- [ ] 合并方式选 **Create a merge commit**（成员自己提 PR 时）
- [ ] 在 [团队网格搜索任务.md](团队网格搜索任务.md) 更新该任务状态

汇总全部结果：

```bash
python scripts/merge_grid_results.py
```

按 `valid_ndcg10` 选取 Top1 超参，再由维护者跑 `01_full_train.ipynb` 全量训练。

## 7. 常见问题

### Contributors 里看不到成员

- commit 邮箱是否与 GitHub 账号绑定的邮箱一致？
- 是否用了 **Squash and merge**（作者变成维护者）？
- 是否尚未合并到 `main`？（仅开 PR 未 merge 不会计入）

### `git push` 被拒绝

- 是否 push 到了自己的 Fork，而不是直接 push 上游（无写权限时）？
- 可先 Fork，或请维护者添加 Collaborator。

### 成员不会用 Git

使用 **方式 B**，维护者 commit 时加 **Co-authored-by**（见第 4 节）。

## 8. 发给成员的一页速查

```text
1. Fork https://github.com/kamineayaka/UserBehavior-SASRec
2. git config user.email 设为 GitHub 已验证邮箱
3. csv/jsonl 放入 results/grid_search/（按命名规范）
4. 新分支 → commit → push → 向 main 提 PR
5. 维护者用 Create a merge commit 合并 → 你出现在 Contributors
```
