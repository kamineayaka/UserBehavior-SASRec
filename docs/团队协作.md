## 二、统一 FAST 配置

在 `notebooks/02_grid_search.ipynb` 中保持：

```python
FAST_MODE = True
fast_num_epochs = 3
fast_eval_user_limit = 100000
eval_k = 10
seed = 42
num_workers = 0
weight_decay = 0.0
# grid_space 内 eval_num_neg 为 [100]
```

**禁止**：`FAST_MODE = False`（全量复核仅维护者执行）。

## 四、成员操作步骤

1. Clone 仓库并准备 `data/` 下 4 个 parquet（见 [使用指南.md](使用指南.md)）
2. 打开 `notebooks/02_grid_search.ipynb`，运行路径 cell 与 memmap 准备 cell
3. 将 `grid_space` 换成认领任务的配置
4. 运行实验循环 cell
5. 将生成的 csv/jsonl **重命名**后放入 `results/grid_search/`
6. 按下方「结果提交」提 PR 或发给维护者

---

## 五、结果提交

### 提交什么

将重命名后的两个文件放入 **`results/grid_search/`**（不要修改 `history/`）：

```
grid_search_fast_{姓名}_{任务ID}_{YYYYMMDD_HHMMSS}.csv
grid_search_fast_{姓名}_{任务ID}_{YYYYMMDD_HHMMSS}.jsonl
```

示例：`grid_search_fast_张三_B1_20260604_153000.csv`

**不要**提交：`data/*.parquet`、`.pt`、`memmap_cache/` 等大文件。

> 任务 ID 请使用 `A1`、`B1` 等标准格式（早期个别文件可能写成 `A`，新提交请统一用 `A1`）。

### 方式 A：Fork + PR（推荐）

1. Fork <https://github.com/kamineayaka/UserBehavior-SASRec>
2. 配置 Git（邮箱须为 GitHub 已验证邮箱）：

```bash
git config user.name "张三"
git config user.email "zhangsan@example.com"
```

3. Clone、放文件、推送分支：

```bash
git clone https://github.com/你的用户名/UserBehavior-SASRec.git
cd UserBehavior-SASRec
git checkout -b grid-search-张三-B1
git add results/grid_search/grid_search_fast_张三_B1_*.csv
git add results/grid_search/grid_search_fast_张三_B1_*.jsonl
git commit -m "grid search: B1 by 张三"
git push origin grid-search-张三-B1
```

4. 向 `main` 提 PR，标题示例：`grid search: B1 (张三)`
5. 维护者用 **Create a merge commit** 合并（保留 Contributors 署名）

| 合并方式 | Contributors 是否显示成员 |
|----------|---------------------------|
| **Create a merge commit**（推荐） | 是 |
| Rebase and merge | 是 |
| Squash and merge | 通常只显示维护者 |

### 方式 B：共享盘 / 群内发给维护者

维护者放入目录后自行 commit。默认不会在 Contributors 显示成员，可在 commit message 加：

```
Co-authored-by: 张三 <zhangsan@example.com>
```

### 一页速查

```text
1. Fork 仓库
2. git config user.email 设为 GitHub 已验证邮箱
3. csv/jsonl 放入 results/grid_search/（按命名规范）
4. 新分支 → commit → push → 向 main 提 PR
5. 维护者用 Create a merge commit 合并
```
