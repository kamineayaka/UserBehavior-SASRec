# 使用指南

按顺序完成以下步骤，即可在本机复现 SASRec 训练与评估。

---

## 1. 获取代码

```bash
git clone https://github.com/kamineayaka/UserBehavior-SASRec.git
cd UserBehavior-SASRec
```

本仓库已包含 `sasrec_core/`。Notebook 会自动将项目根目录加入 `sys.path`，无需依赖上级 monorepo。

---

## 2. 安装依赖

```bash
pip install -r requirements.txt
```

**GPU（推荐）**：先按 [PyTorch 官网](https://pytorch.org/get-started/locally/) 安装与你 CUDA 版本匹配的 `torch`，再安装其余依赖。

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```

---

## 3. 准备数据

训练 parquet **不入 Git**，需从 [GitHub Release](https://github.com/kamineayaka/UserBehavior-SASRec/releases) 获取。

### 方式 A：自动下载（推荐）

需安装 [GitHub CLI](https://cli.github.com/) 并完成 `gh auth login`。

```bash
python scripts/download_release_assets.py
```

覆盖已有文件：`python scripts/download_release_assets.py --force`

### 方式 B：手动下载

1. 打开 [Latest Release](https://github.com/kamineayaka/UserBehavior-SASRec/releases)
2. 下载 4 个 parquet（或 zip 后解压）
3. 放到 `data/` 根目录（与 `data/README.md` 同级）

| 文件 | 约大小 | 说明 |
|------|--------|------|
| `train.parquet` | ~377 MB | 训练序列 |
| `valid.parquet` | ~9 MB | 验证目标 |
| `test.parquet` | ~9 MB | 测试目标 |
| `item2idx_mapping.parquet` | ~15 MB | 物品 ID 映射 |

**注意**：若解压后文件在子目录内，请移动到 `data/` 根目录，不要多留一层目录。

### 可选：Baseline 模型权重（~900 MB）

仅 `03_goal_check.ipynb` 对比 MostPop / ItemCF 时需要；网格搜索与从零训练**不依赖**此文件。

| 文件 | 放置路径 |
|------|----------|
| `baseline_sasrec_20260425_130830_hr10_0.8475_ndcg10_0.7736.pt` | `data/baseline/` |

参考指标 JSON 已在 Git：`data/baseline/baseline_sasrec_20260425_130830.json`。

### 校验

```bash
python -c "from pathlib import Path; d=Path('data'); print([p.name for p in sorted(d.glob('*.parquet'))])"
```

应输出 4 个文件名。字段说明见 [data/README.md](../data/README.md)。

---

## 4. 全量训练

1. 启动 Jupyter，工作目录设为项目根目录或 `notebooks/` 均可
2. 打开 `notebooks/01_full_train.ipynb`
3. 依次运行所有单元

**流程概要**：

1. 解析 `SASREC_DIR` / `cache_dir`（`data/`）
2. 校验 parquet 是否存在
3. （可选）删除旧 `data/memmap_cache/` 与旧模型
4. `build_memmap_cache` → 生成 `data/memmap_cache/`
5. `SASRecEstimator.fit(...)` 训练
6. `evaluate` 输出 HR@10、NDCG@10
7. 保存 `data/sasrec_full_memmap.pt`

**耗时参考**：GPU 上全量 8 epoch 可能需要数小时；CPU 会自动减少 epoch，仅适合冒烟测试。

**与 Baseline 对比**：见 `data/baseline/baseline_sasrec_20260425_130830.json` 的 `metrics` 字段（全量约 98.6 万用户，HR@10 / NDCG@10 约 **0.847 / 0.774**）。

---

## 5. 网格搜索（团队成员）

任务清单与提交方式见 [团队协作.md](团队协作.md)。

打开 `notebooks/02_grid_search.ipynb`：

- 保持 **`FAST_MODE = True`**（3 epoch，`eval_user_limit=100000`）
- 按分配的任务 ID 设置 `grid_space`
- 将输出重命名为 `grid_search_fast_{姓名}_{任务ID}_{时间}.csv/jsonl`，放入 `results/grid_search/`
- **不要**自行设置 `FAST_MODE=False` 跑全量

**维护者**：汇总 `python scripts/merge_grid_results.py`，选定最优超参后运行 `notebooks/04_full_train_B7.ipynb` 做全量训练。

**指标说明**：FAST 结果为 **10 万用户** 抽样评估，数值通常高于 baseline 全量指标，**不可直接比大小**。

- 对比 **SASRec vs MostPop**

| `data/*.parquet` | 输入切分数据（不入 Git） |

## 8. 常见问题

### `ModuleNotFoundError: No module named 'sasrec_core'`

- 确认 `sasrec_core/` 目录存在
- 先运行 notebook 第一个 code cell，确认 `SASREC_DIR` 指向项目根目录
- 若在 monorepo 开发后更新了根目录包：`python scripts/sync_sasrec_core.py --force`

### `FileNotFoundError: 缺少训练输入文件`

运行 `python scripts/download_release_assets.py`，或按上文手动下载 parquet。

### CUDA OOM（显存不足）

在 notebook 中调小 `batch_size`、`maxlen`、`hidden_units`、`num_blocks`，或设置 `eval_user_limit=50000`。

### 指标明显低于 Baseline

- 是否只跑了 1 个 epoch 或使用了 CPU 默认少轮次？
- `eval_user_limit` 是否过小导致波动？
- 是否误删/未重建 `memmap_cache`？

### checkpoint 中无 memmap_dir（03 notebook）

使用 `01_full_train.ipynb` 训练保存的 `sasrec_full_memmap.pt`；旧 checkpoint 可能不兼容。
