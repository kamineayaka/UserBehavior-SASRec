代码级实现细节见 [sasrec_core/SASREC_原理与实现.md](../sasrec_core/SASREC_原理与实现.md)。

## 一、前置基础

### Python 与依赖

- 建议 Python 3.10+，用 `venv` 或 Conda 隔离环境
- 安装：`pip install -r requirements.txt`
- GPU 训练需单独安装带 CUDA 的 [PyTorch](https://pytorch.org/get-started/locally/)

### 常用库

| 库 | 用途 |
|----|------|
| NumPy | 序列数组、memmap 缓存 |
| Pandas | 读取 parquet、查看训练日志 |
| PyArrow | parquet 读写 |
| PyTorch | 模型训练与推理 |

### PyTorch 最小概念

- **Tensor**：模型输入输出；`device="cuda"` 表示用 GPU
- **DataLoader**：按 batch 取训练样本
- **state_dict**：模型权重，由 `SASRecEstimator.save/load` 持久化

需理解 **batch_size、epoch、学习率** 对训练时间与显存的影响。

### 推荐系统基础

- **协同过滤（CF）**：根据用户/物品共现做相似推荐
- **流行度（MostPop）**：推荐全局最热物品，常用弱基线
- **序列推荐**：关注交互**先后顺序**，预测下一步行为

### 评估指标：HR@K 与 NDCG@K

本项目使用**采样排序评估**（1 个正样本 + 若干负样本）：

- **HR@K**：真实下一物品是否出现在 Top-K 中
- **NDCG@K**：若命中，排名越靠前得分越高

两者越高越好。评估有随机负采样波动，固定 `seed` 可提升可复现性。

### 数据切分（Leave-one-out 风格）

每个用户通常有：

- **train**：历史行为序列
- **valid**：验证目标（下一跳）
- **test**：测试目标（valid 之后的下一跳）

本仓库 parquet 已做好切分，无需从原始日志重切。字段说明见 [data/README.md](../data/README.md)。

## 二、SASRec 简介

### 要解决什么问题？

给定用户按时间排序的交互序列 \(S_u = [i_1, i_2, \ldots, i_T]\)，预测**下一个物品** \(i_{T+1}\)。

SASRec（*Self-Attentive Sequential Recommendation*）用 **Transformer 自注意力** 编码历史，使每个位置都能关注序列中任意历史位置（工程上通过 `maxlen` 截断控制成本）。

### 训练时在学什么？

对每个时间步，用**当前前缀**预测**下一物品**：

- **正样本**：真实的下一物品 embedding
- **负样本**：从全物品空间随机采样（避开用户历史）
- 模型输出与正负 embedding 的**点积分数**，用二分类交叉熵优化

### 与本项目代码的对应

| 概念 | 代码位置 |
|------|----------|
| 超参数 | `SASRecConfig`（`config.py`） |
| 模型结构 | `SASRec`（`model.py`） |
| 数据 / memmap | `data.py` |
| 训练与评估 | `trainer.py` |
| 一站式 API | `SASRecEstimator`（`estimator.py`） |

百万级用户推荐使用 **memmap 模式**（`build_memmap_cache` + `fit(input_mode="memmap")`），避免把全量 `user_train` 字典载入内存。

### 参考效果（Baseline）

在约 **98.6 万用户** 的全量数据上（`data/baseline/baseline_sasrec_20260425_130830.json`）：

| 集合 | HR@10 | NDCG@10 |
|------|-------|---------|
| Valid | ≈ 0.8476 | ≈ 0.7747 |
| Test  | ≈ 0.8475 | ≈ 0.7736 |

复现时因硬件、随机种子、训练轮次不同会有小幅波动；显著偏低请查 [使用指南.md](使用指南.md) 排障章节。
